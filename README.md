# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* BMI 계산과 기초 대사량 계산기

<br>


## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 이용자의 이름, 성별, 키, 몸무게, 나이를 입력 받아 이용자의 BMI를 계산해주고 이후 계산한 BMI를 토대로 예상 기초 대사량을 제시해주는 프로그램입니다.
* 이용자는 BMI와 기초 대사량(BMR)에 사용될 정보(이름, 성별, 키, 몸무게, 나이)를 프로그램에 입력합니다. 시스탬은 받은 이용자의 정보를 사용하여 이용자의 BMI를 계산합니다. 그리고 계산된 BMI를 사용하여 이용자의 예상 기초 대사량을 계산하여 이용자에게 제공합니다. 최종적으로는 이용자가 정보를 한 번 입력한 이후 바로 프로그램이 끝나지 않고 이용자가 원할 때 종료가 되고 계산한 BMI수치를 이용해 현재 이용자가 정상인지 저체중인지 비만인지를 판별해내고 이를 근거로 이용자에게 해줄 수 있는 약간의 관리법을 소개해주는 프로그램으로 발전시키는 것이 목표입니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 이름
2. 성별
3. 나이
4. 키
5. 몸무게
6. BMI
7. BMR

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 이용자에게 이용자의 개인정보(이름, 성별, 나이, 키, 몸무게)를 입력받고 받은 정보를 사용해서 BMI와 BMR을 계산하도록 했습니다.
  * 캐릭터의 이니셜(문자), 초기 체력/마나(정수), 기본 공격력(실수)을 입력받는 화면을 구현했습니다.
  * 사칙연산을 활용해 '총합 전투 자원'을 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** 입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수 순서로 진행할 수 있게 로드맵 제작을 요청함.
    * **적용 내용:** 입출력 데이터 다지기(1차) -> 조건문 써서 계산식 구별(2차) -> 반복하기(3차) -> 동시에 여러 데이터 관리(4차)의 흐름으로 프로그램 제작을 하기로 결정함.
      
 2. **내용 2**
    * **프롬프트 요약:** 1차때 의미 있는 변수 5개 이상, 자료형 3개 이상(int, float/double, char 등) 혼합, 입출력 함수 사용(printf, scanf), 산술연산 1개 이상 사용해야하고 조건문(if, switch)이나 반복문(while, for), 함수등 절대 사용하면 안되고 순차구조여야하는 틀을 어떻게 잡을까?
    * **적용 내용:** 아직까지 성별을 받지 못하기 때문에 일단 BMR은 남성을 기준으로 계산을 진행하고 int, char, double을 사용해 이름, 성별, 나이, 키, 몸무게를 변수로 저장하기로 결정.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 성별을 입력받는 문구를 띄우는 과정에서 오류발생
     * **해결:** AI에게 질문하여 뒷부분의 "가 '로 오타가 나 있음을 발견하고 수정
  2. **문제 2:** scanf과정에서 성별을 입력받지 않고 바로 넘어가는 오류 발생
     * **해결:** AI에게 질문하여 %c 앞에 띄어쓰기 추가하며 오류 수정
  3. **문제 3:** error: expected expression before ‘%’ token 라는 오류 발생
     * **해결**: AI에게 질문하여 %s, %c 양 옆에 ""를 붙여야함을 확인받고 수정.
